화살표함수로 선언하고 그안에 options = 객체선언
이거할때 메소드 넣어주고 headers에 어떤식으로 파일을 받을지(accept), 그리고 명세서(Authorization) 넣어주고 bearer넣기(제일중요함)
'Bearer ${token}' 권한(api 액세스 토큰)이 있어야 실행됌.

 const response = await fetch(URL, options);
이 줄은 **"택배 기사님(fetch)에게 주문서(options)를 들려 보내고, 물건(response)이 올 때까지 기다리는 것"**입니다.

① fetch('...')
역할: 브라우저에 내장된 함수로, 지정된 주소(URL)로 네트워크 요청을 보냅니다.
2. return response.json();
이 줄은 **"받은 택배 상자(response)를 뜯어서 내용물(JSON)만 꺼내기"**입니다.

① response.json()
문제점: 서버가 보낸 데이터는 네트워크를 타고 오느라 '문자열'이나 '바이너리' 형태로 뭉쳐져 있습니다. 자바스크립트는 이걸 바로 사용할 수 없습니다.

해결: .json() 함수를 실행하면, 이 복잡한 뭉치를 자바스크립트가 바로 이해할 수 있는 객체(Object)나 배열(Array) 형태로 예쁘게 조립해 줍니다.

특이사항: 이 변환 과정도 시간이 걸리기 때문에 내부적으로는 비동기로 작동합니다.

fetch(메뉴, 주문서): 웨이터(fetch)에게 메뉴(URL)와 내 카드(options)를 줍니다.

await: 요리가 나올 때까지 테이블에서 기다립니다. (1. async (함수 앞에 붙이는 선언문) 2. await (작업 앞에 붙이는 실행문))

response: 요리가 쟁반에 담겨 나왔습니다. (하지만 아직 뚜껑이 덮여 있죠.)

response.json(): 요리의 뚜껑을 열고 먹기 좋게 세팅합니다.

return: 드디어 맛있는 영화 데이터 요리를 내 입(컴포넌트)으로 가져갑니다.




++++ options를 빼고 fetch하면 생기는일
1. 401 Unauthorized (인증 실패) 에러 발생
TMDB 같은 공개 API 서버는 아무에게나 데이터를 주지 않습니다. 반드시 **"이 사람은 허가된 사용자다"**라는 증표(토큰)가 필요한데, options 안에 있는 Authorization 헤더가 그 역할을 합니다.

결과: 서버는 요청을 받자마자 **"누구세요? 신분증이 없으시네요."**라며 401 에러를 던지고 대화를 거부합니다.

브라우저 콘솔: GET https://api.themoviedb.org/... 401 (Unauthorized)라는 빨간색 문구가 뜹니다.

2. 기본값(Default)으로 동작 시도
fetch의 두 번째 인자인 options를 생략하면, 브라우저는 다음과 같은 기본 설정으로 요청을 보냅니다.

Method: 자동으로 GET으로 설정됩니다. (이건 다행히 TMDB 요구사항과 맞습니다.)

Headers: 아무런 정보도 담기지 않습니다. (accept 정보도, Authorization 정보도 없습니다.)

3. 코드의 흐름이 깨짐 (에러의 연쇄 작용)
options가 없으면 다음과 같은 **'도미노 현상'**이 일어납니다.

fetch 실행: 서버로 요청은 가지만, 신분증이 없어 거절당함.

res.json(): 서버가 보낸 것이 영화 데이터가 아니라 "넌 권한이 없어"라는 짧은 에러 메시지이거나 빈 값일 수 있습니다.

setMovies(data.results): data 안에 results라는 목록이 아예 없으므로, 여기서 **"Cannot read properties of undefined (reading 'results')"**라는 치명적인 자바스크립트 에러가 나면서 화면이 하얗게 변할(White-out) 수 있습니다.

4. 비유하자면? (식당 상황)
options 포함: "여기 제 예약권(Token) 있고요, 메뉴는 코스 요리(JSON)로 주세요." -> 식사 성공!

options 제외: (예약 전용 식당에) 그냥 들어가서 식탁에 앉음. -> 웨이터가 "예약 손님만 가능합니다. 나가주세요"라며 쫓아냄.
